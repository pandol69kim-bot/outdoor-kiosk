# 작업내용 - 도매처 로그인 / 배송처리 기능 추가

**날짜:** 2026-06-01  
**작업 범위:** 도매처 전용 로그인, 주문 조회, 배송 상태 업데이트

---

## 작업 개요

기존 관리자 전용 구조에 더해,
도매처가 직접 로그인해서 자기 주문만 확인하고 배송 상태를 업데이트할 수 있는 기능을 추가함.

현재 구현 방식은 `Supabase Auth 계정 이메일`과 `도매처 관리에 등록된 이메일`을 매칭하는 구조임.

---

## 추가된 주요 기능

### 1. 도매처 로그인 화면
- 경로: `/wholesaler/login`
- Supabase 이메일/비밀번호 로그인 사용
- 로그인 후 해당 이메일이 도매처 테이블에 등록되어 있지 않으면 접근 차단

### 2. 도매처 전용 포털
- 경로: `/wholesaler/orders`
- 로그인한 도매처의 주문만 조회
- 주문 상태를 직접 변경 가능
- 허용 상태: `배송중`, `배송 완료`

### 3. 도매처 전용 인증/권한 처리
- `middleware.ts`에서 `/wholesaler/*` 경로 보호
- 로그인 안 된 경우 `/wholesaler/login`으로 이동
- 도매처 이메일과 연결되지 않은 계정은 도매처 포털 접근 불가

### 4. 배송 상태 업데이트 API
- 도매처 본인 주문만 업데이트 가능
- 다른 도매처 주문 수정 불가
- 허용 상태값만 변경 가능

## 생성/수정 파일

### 신규 파일
- `lib/wholesaler-auth.ts` — 로그인 사용자 이메일과 도매처 매핑 확인
- `components/wholesaler/WholesalerNav.tsx` — 도매처 포털 사이드바
- `app/wholesaler/layout.tsx` — 도매처 레이아웃
- `app/wholesaler/page.tsx` — `/wholesaler/orders` 리다이렉트
- `app/wholesaler/login/page.tsx` — 도매처 로그인 화면
- `app/wholesaler/orders/page.tsx` — 도매처 주문 목록 / 배송 상태 변경 화면
- `app/api/wholesaler/orders/route.ts` — 도매처 본인 주문 조회 API
- `app/api/wholesaler/orders/[id]/route.ts` — 도매처 배송 상태 업데이트 API

### 수정 파일
- `middleware.ts` — `/wholesaler/*` 인증 보호 추가
- `components/admin/WholesalerForm.tsx` — 도매처 이메일이 로그인용임을 안내
- `배포가이드.md` — 도매처 로그인 계정 생성 절차 추가

---

## 동작 방식

1. 관리자 페이지에서 도매처 이메일 입력
2. Supabase Authentication에서 같은 이메일로 계정 생성
3. 도매처가 `/wholesaler/login` 로그인
4. 로그인 이메일과 도매처 이메일이 일치하면 포털 접근 허용
5. `/wholesaler/orders`에서 자기 주문만 조회
6. 배송 상태를 `배송중` 또는 `배송 완료`로 변경

---

## 운영 전 준비사항

### 필수
- 도매처마다 `이메일` 값이 정확히 입력되어 있어야 함
- Supabase Authentication에 동일 이메일 계정을 생성해야 함

### 주의
- 현재는 이메일 매칭 기반 권한 처리임
- 즉, 도매처 식별은 별도 사용자 역할 테이블이 아니라 `wholesalers.email` 기준임
- 이메일이 바뀌면 도매처 정보와 로그인 계정을 같이 맞춰야 함
