# 야외 무인 유통 주문 플랫폼 - 배포 가이드

## 1. Supabase 설정

### 1.1 프로젝트 생성
1. [supabase.com](https://supabase.com) 접속 → 무료 계정 생성
2. **New Project** 클릭 → 프로젝트명, DB 비밀번호 설정

### 1.2 데이터베이스 스키마 적용
1. Supabase 대시보드 → **SQL Editor** 탭
2. `supabase/schema.sql` 파일 내용 전체 복사 후 붙여넣기
3. **RUN** 버튼 클릭

### 1.3 Storage 버킷 생성
1. 대시보드 → **Storage** 탭
2. **New bucket** 클릭
3. 이름: `product-images`, **Public** 체크박스 활성화
4. **Create bucket**

### 1.4 관리자 계정 생성
1. 대시보드 → **Authentication** → **Users** 탭
2. **Invite user** 클릭 → 이메일 입력 (이메일로 링크 발송됨)
3. 또는 **Add user** → 이메일 + 임시 비밀번호 직접 설정

### 1.4.1 도매처 로그인 계정 생성
1. 도매처 관리에 등록한 `이메일`과 동일한 주소로 **Authentication → Users** 에서 계정 생성
2. 생성된 계정으로 `/wholesaler/login` 에 로그인
3. 로그인한 도매처는 자기 주문만 조회하고 배송 상태를 업데이트할 수 있음

### 1.4.1 도매처 로그인 계정 생성
1. 도매처 관리에서 등록한 `이메일`과 동일한 주소로 **Authentication → Users** 에서 계정 생성
2. 생성된 계정으로 `/wholesaler/login` 에 로그인 가능
3. 로그인 후 해당 이메일과 연결된 도매처 주문만 조회되고 배송 상태를 업데이트할 수 있음

### 1.5 API 키 확인
- 대시보드 → **Settings** → **API**
- `Project URL` → `NEXT_PUBLIC_SUPABASE_URL`
- `anon public` → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `service_role` → `SUPABASE_SERVICE_ROLE_KEY` (절대 노출 금지)

## 2. Resend 이메일 설정

1. [resend.com](https://resend.com) 가입 (무료 3,000건/월)
2. **API Keys** → **Create API Key**
3. 발급된 키 → `RESEND_API_KEY`
4. **Domains** → 도메인 등록 후 DNS 설정 (또는 초기엔 Resend 기본 도메인 사용)
5. `EMAIL_FROM` = `onboarding@resend.dev` (초기 테스트용)
6. `gmail.com` 같은 개인 메일 주소는 발신자(`from`)로 바로 사용할 수 없음. 자체 도메인을 검증한 뒤 해당 도메인 주소를 사용해야 함

## 3. Vercel 배포

### 3.1 GitHub 업로드
```bash
# GitHub에서 새 리포지토리 생성 후
cd outdoor-kiosk
git remote add origin https://github.com/your-username/outdoor-kiosk.git
git push -u origin master
```

### 3.2 Vercel 배포
1. [vercel.com](https://vercel.com) 가입 (GitHub 로그인 권장)
2. **Add New Project** → GitHub 리포지토리 선택
3. Framework: **Next.js** 자동 감지
4. **Environment Variables** 탭에서 아래 변수 입력:

| 변수명 | 값 |
|--------|-----|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase Project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase anon key |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service role key |
| `RESEND_API_KEY` | Resend API key |
| `EMAIL_FROM` | 발신 이메일 주소 |

5. **Deploy** 클릭 → 약 2분 후 배포 완료
6. Vercel이 `*.vercel.app` 도메인 자동 제공

### 3.3 Vercel에서 GitHub 저장소가 안 보일 때

에러 예시:

- `No Results Found`
- `Your search for "outdoor-kiosk" did not return any results.`
- `Make sure to grant Vercel access to the Git repositories you’d like to import.`

이 경우는 대부분 저장소가 없는 것이 아니라,
**Vercel이 GitHub 저장소 접근 권한을 아직 받지 못한 상태**입니다.

#### Vercel 권한 열기 단계

`Settings → Git` 메뉴가 안 보이는 것이 정상일 수 있습니다.
이 메뉴는 **이미 생성된 Vercel 프로젝트 내부의 Settings** 에서만 보입니다.
처음 Import 하는 단계라면 아래처럼 **GitHub App 권한 화면**에서 확인해야 합니다.

`There aren't any GitHub Apps installed on this repository.` 라는 문구가 보이면,
현재 **저장소 Repository Settings 화면**에서 확인하고 있을 가능성이 큽니다.
이 문구는 해당 저장소에 직접 표시할 GitHub App이 없다는 뜻일 뿐이고,
Vercel 연결 확인은 보통 **계정/조직 기준 Installed GitHub Apps** 화면에서 해야 합니다.

1. Vercel 대시보드에 로그인
2. **Add New...** → **Project** 로 이동
3. GitHub 연동 화면이 나오면 GitHub 계정 연결 상태 확인
4. 저장소가 안 보이면 GitHub에서 Vercel App 권한 화면으로 이동
5. GitHub 우측 상단 프로필 → **Settings** → **Applications** → **Installed GitHub Apps**
6. 목록에서 **Vercel** 선택 후 **Configure** 클릭
7. `Repository access` 에서 아래 둘 중 하나 선택
    - `All repositories`
    - `Only select repositories` 선택 후 `outdoor-kiosk` 체크
8. 저장 후 Vercel의 Import 화면으로 돌아오기
9. Import 페이지 새로고침 후 `outdoor-kiosk` 다시 검색

#### GitHub에 Vercel App이 아예 안 보일 때

1. GitHub 우측 상단 프로필 → **Settings** → **Applications** → **Installed GitHub Apps** 로 이동
2. 목록에 **Vercel** 이 없으면 아직 설치되지 않은 상태
3. Vercel에서 다시 **Add New...** → **Project** 로 들어가 GitHub 연동 시작
4. 또는 GitHub Marketplace / Vercel 연동 화면에서 **Vercel for GitHub** 설치 진행
5. 설치 시 `All repositories` 또는 `outdoor-kiosk` 포함하도록 권한 부여
6. 설치 완료 후 Vercel Import 화면으로 돌아와 저장소 재검색

#### 이미 프로젝트를 만든 뒤 연결 저장소를 확인하는 경로

기존 프로젝트가 이미 하나 만들어져 있다면 그때는 아래 경로로 확인합니다.

1. Vercel 대시보드에서 해당 프로젝트 선택
2. **Settings** 탭 이동
3. **Git** 섹션 확인

즉, `Settings → Git` 은 **전역 계정 메뉴가 아니라 프로젝트 내부 설정 메뉴**입니다.

#### 그래도 안 보일 때 추가 확인

1. Vercel에 연결한 GitHub 계정이 실제 저장소 소유 계정과 같은지 확인
2. 저장소 생성 직후라면 브라우저 새로고침 후 다시 검색
3. 필요하면 GitHub App 권한을 다시 저장하거나 재연결

### 3.4 GitHub 저장소 점검 체크리스트

Vercel에서 저장소가 검색되지 않을 때 아래 항목을 순서대로 확인합니다.

#### 기본 체크

- [ ] GitHub에 저장소가 실제로 생성되어 있음
- [ ] 저장소 이름이 정확히 `outdoor-kiosk` 임
- [ ] 기본 브랜치가 `main` 또는 사용 중인 브랜치로 존재함
- [ ] 로컬에서 `git remote -v` 시 올바른 GitHub 주소가 연결되어 있음
- [ ] 로컬에서 `git ls-remote origin` 시 원격 응답이 정상적으로 나옴

#### GitHub 권한 체크

- [ ] Vercel에 연결된 GitHub 계정이 `pandol69kim-bot` 계정이 맞음
- [ ] GitHub App 권한에서 `outdoor-kiosk` 저장소 접근이 허용됨
- [ ] 조직 저장소라면 조직 권한까지 승인됨
- [ ] 저장소가 Private인 경우에도 Vercel App 접근 권한이 포함되어 있음

#### 최종 체크

- [ ] Vercel Import 화면 새로고침 완료
- [ ] 검색어를 `outdoor-kiosk` 또는 전체 저장소 목록으로 재확인
- [ ] 그래도 안 되면 GitHub App 권한 재설정 후 다시 시도

#### 로컬 확인 명령어

```bash
git remote -v
git branch --show-current
git ls-remote origin
```

이 명령들이 정상 응답하면, 대부분 문제는 Git 저장소가 아니라 **Vercel 권한 설정**입니다.

## 4. 키오스크 태블릿 설정

배포된 URL을 태블릿에서 전체화면으로 실행:

### Android 태블릿
- Chrome 실행 → URL 입력 → 메뉴(⋮) → **전체화면** 또는 **홈 화면에 추가**
- 화면 꺼짐 방지: 설정 → 디스플레이 → 화면 자동 꺼짐 → 끄지 않음

### iOS 태블릿 (iPad)
- Safari 실행 → URL 입력 → 공유(□↑) → **홈 화면에 추가**
- 홈 화면 아이콘 클릭 시 전체화면 앱으로 실행

### 자동 새로고침
앱 자체적으로 3분 비활동 시 자동 메인 화면 리셋 기능 내장.

## 7. 주문 흐름 요약

```
소비자 주문
    ↓
POST /api/orders
    ↓
상품/도매처 정보 스냅샷 저장
    ↓
도매처 notify_type = 'email'?
  ├─ YES → Resend 이메일 자동 발송 → 상태: forwarded
  └─ NO  → 상태: received → 관리자가 수동으로 알림발송 버튼 클릭
```
